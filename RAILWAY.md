# Як зробити, щоб список не пропадав після перезапуску Railway

У Railway **Volumes** (постійний диск) налаштовуються **не в Settings**. Ось де шукати.

---

## Де знайти Volumes

### Спосіб 1. Командна палітра (найпростіше)

1. Відкрий свій проєкт на [railway.app](https://railway.app).
2. Натисни **Ctrl+K** (Windows/Linux) або **Cmd+K** (Mac) — відкриється пошук по проєкту.
3. Введи **Volume** (або **Add Volume**).
4. Обери щось на кшталт **Add Volume** / **Create Volume**.
5. Обери **сервіс** (свій бот).
6. Коли попросить **Mount Path** — можна залишити запропонований (наприклад `/data`) або вписати `/data`.
7. Збережи. Після цього перезапусти деплой (Redeploy), якщо сам не перезапустився.

Бот сам побачить, що є volume (Railway ставить змінну `RAILWAY_VOLUME_MOUNT_PATH`), і буде зберігати `data.json` там. **Ніякі Variables вручну для цього задавати не потрібно.**

---

### Спосіб 2. Клік правою кнопкою по сервісу

1. На головному екрані проєкту бачиш **прямокутники (сервіси)**.
2. Клікни **правою кнопкою миші** по тому прямокутнику, де крутиться твій бот.
3. У контекстному меню шукай пункт типу **Add Volume** / **Volumes** / **New Volume**.
4. Далі вкажи Mount Path (наприклад `/data`) і збережи.

---

### Якщо нічого подібного немає

- Можливо, Volumes є тільки на платних планах (Hobby/Pro). На **безкоштовному** їх іноді не показують або обмежують.
- Тоді варіанти такі:
  - **Нічого не робити** — бот працює, але після кожного редеплою список обнулиться.
  - Або перейти на платний план Railway, якщо потрібне збереження списку.

---

**Підсумок:** спочатку спробуй **Ctrl+K** (або Cmd+K) → ввести **Volume** і створити volume для свого сервісу. Змінні вручну для шляху до файлу ставити не потрібно — бот сам використовує volume, якщо він є.
